## 1. top(Table of Processes)
시스템의 상태와 프로세스들을 **실시간으로 모니터링**하는 명령어임.


<img width="1037" height="236" alt="top사진" src="https://github.com/user-attachments/assets/5c222739-c953-4411-b6a1-e5abc11b46e7" />


### 사용방법
$ top


### 설명
| 출력 | 설명 | 비고 |
| :--- | :--- | :--- |
| PID | 프로세스 ID | |
| USER | 프로세스 소유자 | |
| %CPU | CPU 사용률 | |


| 명령어 | 설명 | 비고 |
| :--- | :--- | :--- |
| 'q' | 나가기 | 명령어 실행 시 | 

## 2. ps (Process Status)
**현재 실행 중인 프로세스의 "스냅샷"을 확인하는 명령어**이며 실시간이 아니라 명령어를 입력한 그 순간의 상태를 보여줌


<img width="466" height="92" alt="ps 사진" src="https://github.com/user-attachments/assets/7090839e-84fe-4cbc-bcaa-c308b3fdb42b" />



### 사용방법
$ ps

### 설명
| 옵션 | 설명 | 비고 |
| :--- | :--- | :--- |
| -e | 시스템의 모든 프로세스 표시 | |
| -f | 풀 포맷(Full format)으로 상세 정보 표시 | |
| -ef | 모든 프로세스를 상세하게 출력 | 가장 많이 쓰는 명령어 |


| 출력 | 설명 | 비고 |
| :--- | :--- | :--- |
| UID | 프로세스 소유자 ID | |
| PID | 프로세스 고유 ID | 이 번호로 kill 명령어를 사용 |
| CMD | 실행된 명령어 이름 |


## 3. jobs
**현재 셸(Shell) 세션에서 실행된 작업(Job)의 목록을 확인하는 명령어**

주로 백그라운드에서 돌고 있는 작업을 확인할 때 사용




### 사용방법
리눅스에서 작업 할 시

$ jobs&


### 설명
| 출력 | 설명 | 비고 |
| :--- | :--- | :--- |
| [1], [2] | 작업 번호 | kill 명령을 쓸 때 사용 |
| + | 현재 작업 | |
| - | 바로 이전 작업 | |


## 4. kill
**특정 프로세스에 "시그널(Signal)"을 보내는 명령어**


주로 **프로세스를 강제 종료**할 때 사용


### 사용방법
$ kill [옵션] [PID]


옵션 안줄 시 PID에게 요청
### 설명
| 옵션 | 설명 | 비고 |
| :--- | :--- | :--- |
| -15 | 프로세스에게 "종료해달라" 요청 | SIGTERM |
| -9 | 프로세스 **강제 종료** | SIGKILL |
| -2 | 인터럽트(=^c) | SIGINT |
